React Hook Form (RHF)을 사용 중이며, 컴포넌트 내에 파일 업로드 기능과 카카오 지도 API와 같은 대용량 비동기 외부 API 연동 기능이 함께 있습니다.

최근 `<지도/주소 기능 추가 또는 이미지 업로드 로직 변경>` 후, `<등록 페이지>`에서 **특정 기능이 오작동**하고 있습니다.

## 🔴 현재 상황 및 문제 증상

1.  **오작동 기능:** `<이미지 다중 업로드 실패, 주소 검색 후 지도가 뜨지 않음 등>`
2.  **특이 사항:** `<등록 모드 ('create')에서만 문제가 발생하며, 수정 모드 ('edit')에서는 문제가 없음.>` 또는 `<특정 Input 필드에 값을 입력하면 다른 Input 필드가 리셋됨.>`
3.  **에러 로그:** `<"카카오 지도 API (services) 라이브러리가 로드되지 않았습니다." 또는 "TypeError: Cannot read properties of undefined (reading 'length')" 등 구체적인 콘솔 에러 메시지>`

## 🔍 핵심 원인 분석 요청

현재 컴포넌트의 복잡성(RHF + 외부 API + 다중 상태)을 고려하여, 아래 두 가지 관점에서 코드를 분석하고 안정적인 해결책을 제시해 주세요.

1.  **상태 불일치 (이미지 문제):**
    * `<이미지 파일>` 관리가 **RHF의 `Controller` 상태**와 **컴포넌트의 `useState` (예: selectedFiles)**로 이원화되어 있는지 확인하고, 이로 인해 잦은 리렌더링 시 **상태 불일치(Desync)**가 발생했는지 진단해 주세요.
    * **해결책은 `DataTransfer`를 사용하여 `Controller`를 유일한 진실의 원천(Single Source of Truth)으로 만드는 것**입니다.

2.  **비동기 타이밍 (지도/API 문제):**
    * `<주소 변환(Geocoding) 등>` 외부 API 호출이 **API가 완전히 로드되기 전에** 실행되었는지 확인해 주세요.
    * **해결책은 API 호출 함수 내부에 **`setTimeout`이나 재시도(Retry) 로직**을 추가하여 API 객체가 로드될 때까지 기다리게 만드는 것**입니다.

분석 후, **수정해야 할 파일의 경로/함수명**과 **구체적인 코드 수정 방안**을 제시해 주세요.
