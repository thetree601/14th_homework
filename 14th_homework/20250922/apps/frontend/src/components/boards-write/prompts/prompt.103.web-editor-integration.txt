==============================================

핵심요구사항) 게시판 등록/수정 페이지에서 내용 입력을 위한 textarea를 react-quill 웹 에디터로 교체할 것.

핵심요구사항) react-quill을 dynamic import를 사용하여 SSR을 방지하고 코드 스플리팅을 적용할 것.

핵심요구사항) react-quill의 onChange 이벤트를 react-hook-form의 setValue와 trigger를 사용하여 폼 상태와 동기화할 것.

핵심요구사항) 등록 모드와 수정 모드 모두에서 웹 에디터가 정상적으로 작동하도록 구현할 것.

구현방법) 
1. react-quill 패키지 설치 (package.json에 추가)
   - "react-quill": "^2.0.0"
   - react-quill v2는 타입 정의가 내장되어 있어 별도의 @types 패키지 불필요

2. react-quill CSS 파일 import
   - "react-quill/dist/quill.snow.css" import 추가

3. dynamic import를 사용하여 ReactQuill 컴포넌트 로드
   - next/dynamic의 dynamic 함수 사용
   - ssr: false 옵션 설정

4. hook.ts에 onChangeContent 함수 추가
   - ReactQuill의 onChange 콜백으로 사용
   - setValue("contents", value)로 폼 값 설정
   - trigger("contents")로 유효성 검사 트리거

5. index.tsx에서 textarea를 ReactQuill 컴포넌트로 교체
   - 기존 textarea 제거
   - ReactQuill 컴포넌트 추가
   - register 대신 onChangeContent 사용
   - value는 watch("contents")로 현재 값 표시

6. 수정 모드에서 기존 내용이 에디터에 표시되도록 처리
   - useEffect에서 reset 시 contents 값이 자동으로 반영됨
   - ReactQuill의 value prop에 watch("contents") 사용

주의사항) 
- ReactQuill은 register를 직접 사용할 수 없으므로 setValue와 trigger를 사용해야 함
- ReactQuill의 onChange는 event 객체가 아닌 value(string)를 직접 전달함
- dynamic import를 사용하여 SSR 오류 방지 (react-quill은 브라우저 전용)
- CSS 파일은 반드시 import해야 에디터 스타일이 적용됨
- 수정 모드에서 기존 내용이 제대로 표시되는지 확인 필요
- 폼 유효성 검사가 정상적으로 작동하는지 확인 필요

예시코드구조:
```typescript
// hook.ts에 추가할 함수
const onChangeContent = (value: string) => {
  console.log(value);
  // register로 등록하지 않고, 강제로 값 넣어주는 방법
  setValue("contents", value);
  trigger("contents");
};

// return 객체에 추가
return {
  // ... 기존 반환값들
  onChangeContent,
};
```

```typescript
// index.tsx 상단에 추가
"use client";

import dynamic from "next/dynamic";
import "react-quill/dist/quill.snow.css";

// 코드 스플리팅 - SSR 방지
const ReactQuill = dynamic(() => import("react-quill"), { ssr: false });
```

```typescript
// index.tsx의 내용 입력 부분 수정
<div className={`${styles.formGroup} ${styles.noBorder}`}>
  <label className={styles.labelRequired}>내용</label>
  <ReactQuill
    value={watch("contents") || ""}
    onChange={onChangeContent}
    placeholder="내용을 입력해 주세요."
    style={{ minHeight: "300px" }}
  />
  {errors.contents && <div className={styles.errorMessage}>{errors.contents.message}</div>}
</div>
```

추가고려사항)
- react-quill의 툴바 옵션을 커스터마이징할 수 있음 (기본값 사용 권장)
- 에디터 높이를 조절하기 위해 style prop 사용
- 에디터와 에러 메시지 사이의 간격 조정 필요할 수 있음
- CSS 모듈에서 에디터 스타일을 추가로 커스터마이징 가능
- 이미지 업로드 기능이 필요한 경우 react-quill의 이미지 핸들러 커스터마이징 필요 (현재는 텍스트만)
